Getting Started with didhelpers: Diagnosing Your DiD Design

Overview

didhelpers provides lightweight diagnostic and visualization utilities for staggered difference-in-differences (DiD) designs. It is not a new estimator — rather, it is the companion toolkit you wish you had when setting up a staggered DiD analysis.

The package offers three core diagnostics and a convenience wrapper:

  1. Treatment timing plot — see when each unit gets treated.
  2. Pre-trends test — event-study regression to check the parallel trends assumption.
  3. Bacon decomposition summary — understand which 2×2 comparisons drive your TWFE estimate.
  4. Full diagnostics report — run everything at once.

1. Plot Treatment Timing

The first step in any staggered DiD analysis is understanding the treatment variation in your panel. plot_treatment_timing() produces a heatmap showing when each unit switches from untreated to treated:

plot_treatment_timing(staggered_data, unit, time, treat)

Units are grouped by treatment cohort on the y-axis. This makes it easy to spot the staggered adoption pattern and identify never-treated units.

2. Test Pre-Trends

The parallel trends assumption is central to any DiD design. test_pretrends() runs an event-study regression with unit and time fixed effects and returns pre-treatment lead coefficients:

pt <- test_pretrends(staggered_data, unit, time, outcome, treat, n_lags = 3)
#> Wald test, H0: joint nullity of lead_3 and lead_2
#>  stat = 0.232853, p-value = 0.792378, on 2 and 400 DoF, VCOV: IID.
pt$coefficients
#> # A tibble: 6 × 6
#>   term   estimate std_error statistic  p_value rel_time
#>   <chr>     <dbl>     <dbl>     <dbl>    <dbl>    <dbl>
#> 1 lead_3 -0.00199     0.134   -0.0149 9.88e- 1       -3
#> 2 lead_2 -0.0917      0.174   -0.528  5.98e- 1       -2
#> 3 lag_0   1.93        0.172   11.2    1.48e-25        0
#> 4 lag_1   1.95        0.175   11.1    3.51e-25        1
#> 5 lag_2   1.84        0.154   11.9    2.56e-28        2
#> 6 lag_3   1.95        0.146   13.4    4.86e-34        3

The joint F-test checks whether all pre-treatment coefficients are jointly zero:

pt$f_test
#> $statistic
#> [1] 0.2328527
#> 
#> $p_value
#> [1] 0.7923775

Since our simulated data has no true pre-trends, the lead coefficients should be small and statistically insignificant. The autoplot() method produces the classic event-study coefficient plot:

autoplot(pt)

You can adjust the confidence level (default is 95%):

autoplot(pt, conf.level = 0.90)

3. Bacon Decomposition

Goodman-Bacon (2021) showed that the two-way fixed effects (TWFE) estimator is a weighted average of all possible 2×2 DD comparisons. Some of these comparisons may use already-treated units as controls, leading to potential bias. bacon_weight_summary() provides a clean summary:

bs <- bacon_weight_summary(staggered_data, unit, time, outcome, treat)
bs$summary
#> # A tibble: 3 × 4
#>   type                     n_comparisons total_weight weighted_avg_estimate
#>   <chr>                            <int>        <dbl>                 <dbl>
#> 1 Earlier vs Later Treated             3        0.229                  2.00
#> 2 Later vs Earlier Treated             3        0.198                  2.02
#> 3 Treated vs Untreated                 3        0.573                  1.89

The scatter plot shows each 2×2 comparison’s estimate against its weight, colored by comparison type:

autoplot(bs)

Input Validation

All core functions validate their inputs and return informative error messages if something is wrong:

# Incorrect column name
test_pretrends(staggered_data, unit, time, outcome, wrong_col)
#> Error: Column(s) not found in `data`: wrong_col

# Non-data-frame input
test_pretrends("not a data frame", unit, time, outcome, treat)
#> Error: `data` must be a data frame.

Relationship to Other Packages

didhelpers is a diagnostic companion, not a replacement for any estimator package. Here is how it relates to the ecosystem:

Package Relationship Notes
bacondecomp Hard dependency bacon_weight_summary() wraps bacondecomp::bacon(), adding S3 class, tidy output, broom methods, and gt/kable tables.
fixest Hard dependency test_pretrends() uses fixest::feols() internally. fixest’s own iplot() is more flexible for fixest-native workflows; didhelpers bundles the event-study + F-test + broom integration into a single call.
HonestDiD Complementary Provides sensitivity analysis for parallel trends violations. No overlap — could be used after test_pretrends() flags concerns.
did2s No overlap Implements the Gardner (2022) imputation estimator. didhelpers diagnoses TWFE; did2s replaces it.
DIDmultiplegt No overlap Suite of robust DiD estimators. didhelpers could diagnose data before running these estimators.
staggered Minimal overlap Provides Callaway–Sant’Anna and Sun–Abraham estimators with their own plot methods. These plot estimates from robust estimators; didhelpers plots TWFE diagnostics.

Next Steps

These diagnostics help you decide whether a standard TWFE estimator is appropriate for your data. If the Bacon decomposition reveals substantial weight on problematic comparisons (e.g., already-treated as controls), or if pre-trends tests fail, consider robust estimators such as those in the DIDmultiplegt family of packages.
